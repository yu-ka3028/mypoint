# mypoint - Claude Code 設定

## セッション開始時

必ず `docs/DESIGN.md` を読んでプロジェクトの文脈を把握すること。

## Plan Mode での挙動

- ソースコードの提案・コードブロックは不要
- 設計の意図・ファイル構成・処理の流れを日本語で簡潔に記述するだけでよい
- サブエージェント（Explore / Plan）の使用は最小限に留める
- トークン節約を最優先する

## 実装方針

- 過剰な抽象化・将来対応は不要、現在の要件のみ実装する
- 変更は最小限に留める
- コメント・ドキュメントは指示がない限り追加しない

## ブランチ運用

- `main`: 常にクリーンな状態を保つ
- `setup/*`: 環境構築
- `feature/*`: 機能実装
- `docs/*`: ドキュメント追加・更新
- 作業前に必ずブランチを確認・作成する

## コミット前の確認

- プッシュ・PR作成前に必ずブランチを確認してユーザーに伝える
